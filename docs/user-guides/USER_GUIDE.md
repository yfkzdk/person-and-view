2. **打断生成**
   ```
   💬 请输入: 请讲一个很长的故事
   ```
   等待2秒后输入：
   ```
   💬 请输入: interrupt

   系统会立即停止生成

3. **暂停和恢复**
   ```
   💬 请输入: pause
   💬 请输入: resume

4. **发送音频**
   ```
   💬 请输入: audio
   ```
   系统会发送模拟音频并显示 VAD 检测结果

5. **退出程序**
   ```
   💬 请输入: quit
   ```

## 常见问题

### Q1: 服务器启动失败？

**检查端口占用**：
```bash
netstat -ano | findstr :8000
```

如果端口被占用，修改 `start_server.bat` 中的端口号：
```bash
--port 8001
```

### Q2: 客户端连接失败？

**确认服务器已启动**：
- 检查服务器终端是否显示 "Application startup complete"
- 确认地址是 `ws://localhost:8000`

### Q3: 没有收到音频？

**检查 ANTHROPIC_API_KEY**：
- 系统需要 API key 才能调用 LLM 和 TTS
- 在 `.env` 文件中设置：
  ```
  ANTHROPIC_API_KEY=your_key_here
  ```

### Q4: 如何使用真实音频？

目前客户端使用模拟音频。要使用真实音频：
1. 安装 PyAudio：`pip install pyaudio`
2. 修改 `client_demo.py` 添加麦克风录制功能
3. 将录制的音频发送到服务器

## 高级用法

### 自定义服务器地址

修改 `client_demo.py` 第11行：
```python
def __init__(self, server_url: str = "ws://localhost:8000/ws/demo-session"):
```

### 修改会话ID

每个会话ID对应独立的会话状态：
```python
client = VoiceNarrativeClient("ws://localhost:8000/ws/my-session")
```

服务器会输出详细日志：
```
INFO:     [Session demo-session] Text input: 你好

```
